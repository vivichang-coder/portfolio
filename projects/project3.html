Complex System Simplification

Redesigning Login Experience for Multi-Server Management

Simplifying Access for Complex Surveillance Systems

Overview

This project focused on redesigning the login experience for a surveillance management platform transitioning from a single-server model to a multi-server architecture.

The goal was to reduce cognitive load, improve login efficiency, and enable users to easily manage and access multiple servers within a unified flow.

Role

UX Researcher / UX Designer (Redesign Owner)

  • Led the redesign phase based on existing usability findings
  • Translated research insights into new UX flows and interaction patterns
  • Designed low-fidelity and high-fidelity prototypes
  • Planned and conducted usability testing (lo-fi → hi-fi)
  • Iterated design based on validation results

Collaboration

  • Previous Research Owner: Provided baseline usability findings
  • PM: Aligned on product direction and multi-server strategy
  • Engineering (RD): Validated technical feasibility and system constraints
  • UI Designer: Refined visual hierarchy and interaction details
Problem & Impact

Problem

  • Existing login flow was designed for single-server access
  • Multi-server functionality introduced complexity that the original UI could not support
  • Login methods (LAN search, history, manual input) were fragmented and inconsistent
  • Key features (e.g. password recovery) were missing
  • Users experienced confusion when switching between different login methods

Resulted in: high cognitive load, inefficient login process, and poor scalability for future features.

Impact

  • Simplified multi-server login experience
  • Improved clarity of system structure (single vs grouped servers)
  • Reduced effort in accessing and managing servers
Approach
1

Insight → Design Mapping

Insight

  • Users struggled to understand multi-server concepts within a single-server UI
  • Fragmented entry points increased decision complexity

Design Decisions

  • Replace dropdown-based selection with structured server list
  • Consolidate all login methods into a single, unified entry point
  • Introduce linear flow for server discovery and management
  • Clearly distinguish single vs grouped servers
2

Solution Design

  • Centralized login entry as a multi-server list
  • Integrated server management directly into login interface
  • Introduced clear categorization (server type, group structure)
  • Enabled automatic LAN discovery on entry
  • Reduced steps and decision points in login flow
3

Validation

Lo-fi Testing (Prototype-based)

  • Recruited same 6 participants from previous study
  • Evaluated usability of new login flow using interactive prototypes

Findings

  • Users clearly understood single vs multi-server concepts
  • Consolidated login entry significantly reduced confusion
  • Linear flow improved task completion efficiency

Hi-fi Testing (Pre-launch)

  • Recruited 10 new participants across experience levels
  • Tested in real environment with high-fidelity UI

Findings

  • Reduced confusion from hidden features and unclear steps
  • Improved feedback and error prevention
  • Overall login experience became more intuitive and efficient
Key Decisions
  • Replace selection-based interaction → structured visibility
  • Replace fragmented entry points → unified flow
  • Replace non-scalable UI → system-oriented architecture
  • Prioritize clarity of system structure over minimal UI
Challenges

Designing for a New Mental Model

  • Users were familiar with single-server logic
  • Multi-server required a new conceptual model

Gradually introduce structure through clear hierarchy and visual grouping

Working with Existing Research

  • Initial usability testing was conducted by another owner

Validated and extended findings through iterative testing (lo-fi → hi-fi)

Balancing Simplicity vs Scalability

  • Simplifying flow vs supporting complex server structures

Design a scalable framework that remains intuitive for new users

Learnings
  • Redesigning flows often requires redefining mental models, not just UI changes
  • Validation across multiple fidelity levels strengthens design decisions
  • A well-structured system can reduce complexity more effectively than simplifying UI alone
  • Even entry points like login can significantly impact overall product experience
